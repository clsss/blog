Dockerfile详解

docker-library

  1. FROM

尽量使用官方的image，因为安全

FROM scratch        # 制作base image
FROM centos         # 使用base image
FROM ubuntu:14.04
  1. LABEL

类似注释

LABEL maintainer="670199384@qq.com"
LABEL version="1.0"
LABEL description="描述"
  1. RUN

执行命令，反斜线换行

RUN yum install && yum install -y vim \
    python-dev

RUN apt-get update && apt-get install -y perl \
    pwgen...
    
RUN /bin/bash -c 'source $HOME/.bashrc;echo $HOME'
  1. WORKDIR

设定当前工作目录 1. 用WORKDIR，不要用RUN cd 2. 尽量使用绝对目录

WORKDIR /test   # 如果没有,会自动创建test目录
WORKDIR demo
RUN pwd         # 输出结果应该是/test/demo
  1. ADD && COPY

添加和复制文件 1. 大部分情况，COPY优于ADD 2. ADD除了COPY还有额外功能（解压） 3. 添加远程文件/目录请使用curl或者wget

ADD hello /
ADD test.tar.gz /   # 添加到根目录并解压

WORKDIR /root
ADD hello test/     # /root/test/hello

WORKDIR /root
COPY hello test/     # /root/test/hello
  1. ENV

多用，可增加可维护性

ENV NODE_ENV product        # 设置环境变量
RUN git push ${NODE_ENV}    # 引用常量
  1. VOLUME 和 EXPOSE
  • 存储
  • EXPOSE（网络端口暴露）
  1. CMD 和 ENTRYPOINT
  • CMD:
    • 设置容器启动后默认执行的命令和参数；
    • docker run指定了其他命令，CMD命令会被忽略；
    • 如果指定了多个CMD，只有最后一个会执行
  • ENTRYPOINT：
    • 设置容器启动时运行的命令
    • 让容器以应用程序或服务的形式运行
    • 不会被忽略，一定会执行
  1. Shell格式和Exec格式
FROM centos
ENV name Docker

# Shell格式
ENTRYPOINT echo "hello $name"
// 输出：hello Docker


# Exec格式
ENTRYPOINT ["/bin/echo", "hello $name"]
// 输出：hello $name
ENTRYPOINT ["/bin/echo", "-c", "echo hello $name"]
// 输出：hello Docker
